# Day14 全链路验收与发版检查清单

更新时间：2026-04-16

## 0. 使用方式

- 建议先执行自动检查：`bash scripts/day14_release_check.sh`
- 开发联调可临时放行：`bash scripts/day14_release_check.sh --allow-pending`
- 自动检查通过后，再按下面 1-6 节做人工验收
- 所有关键项通过后，才进入 App Store 提交流程

---

## 1. 学习链路（首页任务 -> 闯关/对话/每日一词）

- [ ] 首页显示「今日任务卡」，含 4 项任务与总进度
- [ ] 每日一词可正常播放（普通话/东北话），播放中状态正确
- [ ] 完成 3 题闯关后，任务计数正确增加
- [ ] 完成 1 次对话练习后，任务计数正确增加
- [ ] 4 项完成后出现「领取今日奖励」，且当天仅可领一次

通过标准：
- 任务状态刷新一致（重进页面不丢失）
- 不出现“任务已完成但按钮状态未刷新”的异常

---

## 2. 复习链路（错题入库 -> 1/3/7 调度 -> 今日待复习）

- [ ] 闯关答错后，条目进入「易混淆词」
- [ ] 同时生成 1/3/7 天复习计划（可在待复习页观察到）
- [ ] 首页「去复习」可进入「今日待复习」
- [ ] 连刷模式可连续推进，列表模式可单条完成
- [ ] 完成复习后，今日任务“复习一次”计数 +1

通过标准：
- 待复习数量与实际条目一致
- 完成后不会重复出现同一轮任务

---

## 3. 打卡链路（真实性 -> 历史 -> 排行）

- [ ] 打卡提交前能看到真实性预检卡
- [ ] 打卡成功页显示可信度等级和说明
- [ ] 成就墙历史记录可看到可信度标签
- [ ] 排行榜条目可看到可信度标签
- [ ] 点击打卡入口（地图页按钮）可正常进入景点选择

通过标准：
- 同一条打卡记录在成功页、历史、排行榜的可信口径一致

---

## 4. 音频链路（预检 -> 失败原因 -> 缓存命中）

- [ ] 设置页可运行「音频健康检查」并展示建议
- [ ] 设置页展示缓存指标：命中次数、回源次数、命中率、最近清理时间
- [ ] 人为制造失败后，诊断区可显示最近一次失败原因
- [ ] 每日一词播放失败时可见重试入口并可恢复
- [ ] 清理缓存后指标正确归零并记录清理时间

通过标准：
- 失败场景可定位（有原因），可恢复（可重试），可观测（有指标）

---

## 5. 素材合规门禁（风景图）

执行命令：

- 严格门禁：`python3 scripts/verify_scenic_images.py --gate-release`
- 仅报告：`python3 scripts/verify_scenic_images.py`

检查项：

- [ ] `docs/scenic-image-verification-latest.md` 已更新
- [ ] `docs/scenic-image-pending-batch-plan.csv` 已分配负责人与截止时间
- [ ] 发布门禁结果明确显示为「可发布」才可过审
- [ ] 若结果为「阻断发布」，必须先补图并更新 `imageMatchType`

通过标准：
- 严格门禁返回码为 `0`
- 报告中不存在必须阻断项

---

## 6. 发版前最终确认（App Store 提交前）

- [ ] `docs/privacy.html`、`docs/index.html` 与 App 内隐私文案一致
- [ ] App 描述口径保持「核心离线 + 可选联网」
- [ ] 通用设备或模拟器构建通过（无编译错误）
- [ ] 关键手工路径回归通过（本清单 1-5 节）
- [ ] 确认无阻断级问题后再上传构建

---

## 建议执行顺序

1. 先跑自动脚本：`bash scripts/day14_release_check.sh`
   - 若仅验证代码链路可临时用 `--allow-pending`，但不能用于最终发版放行
2. 再做手工冒烟：学习 -> 复习 -> 打卡 -> 音频
3. 最后看素材门禁和文案一致性
4. 全部勾选后提交审核
